# AI Friday Prep Skill

This skill helps generate comprehensive meeting preparation materials for AI Friday sessions.

## When to Use
Use this skill when preparing for an upcoming AI Friday session. It will help you create all the necessary materials, checklists, and communication templates.

## What This Skill Does

### 1. Meeting Preparation Analysis
- Reviews upcoming session details and requirements
- Identifies speakers, topics, and logistics needs
- Checks for potential issues or missing elements

### 2. Material Generation
- **Host checklist**: Pre-meeting setup tasks and tech requirements
- **Speaker prep guide**: Guidelines, timing, and technical requirements
- **Communication templates**: Reminders, announcements, and follow-up messages
- **Backup content**: Baseline topics ready if speakers drop out

### 3. Logistics Coordination
- **Technical setup**: Recording, screen sharing, and tool requirements
- **Timing coordination**: Agenda breakdown with precise timing
- **Contingency planning**: What to do if speakers cancel or tech fails

### 4. Communication Assets
- **Email templates**: Formatted reminders for different audiences
- **Slack/Teams messages**: Quick announcements and updates
- **Calendar descriptions**: Rich meeting details for invites

## Usage Instructions

When you invoke this skill, provide:
- **Session date**: When the meeting is happening
- **Confirmed speakers**: Who's presenting and their topics
- **Session type**: Regular session, special theme, or baseline topics
- **Special requirements**: Any unique needs for this session

The skill will generate a complete prep package including all necessary materials and checklists.

## Output Format

The skill provides organized prep materials including:
- Pre-meeting checklist for hosts
- Speaker communication templates
- Technical setup guide
- Backup plan with baseline topics
- Day-of execution timeline

## Integration

This skill works with:
- Session templates in `meetings/templates/`
- Baseline topics in `content/baseline-topics/`
- Speaker submissions in `content/submissions/`
- Process documentation in `docs/playbooks/`
